    Introduction

    WrapperController

    En este apartado podrás encontrar todo lo relativo al uso de Wrapper Controller, sus funciones y clusters que utiliza de forma interna.

    Esta librería, desarrollada en LabVIEW, hace uso de diversos servicios/librerías gracias a las cuales somos capaces de ofrecer toda la funcionalidad necesaria para que este nuevo sistema sea capaz de sustituir al viejo sistema de RFTag.

    Pinche aquí para obtener más información sobre WrapperController.

    WrapperLibrary

    WrapperLibrary es una librería desarrollada en C# que permite entre otras cosas hacer uso del servicio de QTS, es aquí donde se encapsulan todas las funciones y se simplifica su uso y definición para así ser usable de forma más sencilla sin necesidad de una alta complejidad.

    Help

    Pinche aquí para obtener más información sobre WrapperLibrary.

    Quality Tracking System (QTS)

    QTS, es un servicio en la nube disponible desde un servidor interno de VEP (Valencia Engine Plant). Gracias al cual podemos hacer reportes y consultas sobre la base de datos de QLSCM.

    Es este servicio el encargado de hacer las peticiones sobre la base de datos para subir o consultar todos los datos relativos a la trazabilidad del proceso de mecanizado de piezas.

    Pinche aquí para obtener más información sobre QTS.

    Simple Object Access Protocol (SOAP)

    SOAP es un protocolo estandar que define como dos objetos en diferentes procesos pueden comunicarse mediante el uso de mensajeria XML.
